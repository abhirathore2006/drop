Reference

Deployment

Run Drop on EC2 or Kubernetes — the api and edge behind one ingress, with external managed Postgres (RDS / CloudSQL) and S3.

Two modes

Drop ships in two modes. Static-only (the default everywhere — and the only mode on EC2 / ECS) publishes and serves sites; /v1/apps, /v1/databases and the app-secret routes return 501. Compute (Kubernetes only, opt-in via compute.enabled=true) adds container apps, managed Postgres, and write-only app secrets on top. Everything below is the static-only base; the Compute plane section adds the PaaS.

AWS

Want copy-pasteable AWS CLI commands for every component (ECR, S3, RDS, ACM, Secrets, security groups, ALB, Route 53) with two variants — EKS and ECS Fargate (no Kubernetes)? See AWS deployment, or the Terraform page for the same components as code.

Drop is two processes from one image: a control-plane api and a read-only serving edge. Both sit behind a single ingress. All metadata lives in an external managed Postgres; the served file bytes live in S3 (the chart bundles neither — backups and HA are the platform's responsibility). The api runs schema migrations on boot under a Postgres pg_advisory_lock, so a multi-replica rollout is safe: one pod migrates while the rest wait, then serve. The edge connects read-only and never migrates.

Note

The Helm chart deploys api + edge as Deployments behind one Ingress. For a plain EC2 / VM install, run the same image with the command overridden per process (node dist/api.js / node dist/edge.js) and set the same DROP_* environment variables shown below.

Prerequisites

IMAGE

Image in a registry

Build and push one image (both bundles): podman build -f infra/Dockerfile -t <repo>/drop:<tag> . then podman push to ECR.

TLS

Wildcard TLS

A wildcard certificate for *.<baseDomain> — via ACM (ingress annotations) or a TLS Secret.

IAM

S3 role (IRSA)

An IAM role with read/write on the S3 bucket, referenced from serviceAccount.annotations. No access keys — auth is via the pod's role.

DB

Managed Postgres

RDS / CloudSQL / company DB reachable from the cluster. The chart does not bundle one.

SECRET

Secret with credentials

A Secret holding DROP_GOOGLE_CLIENT_SECRET, DROP_SESSION_SECRET, and DROP_DATABASE_URL (e.g. via External Secrets), or set secret.create=true.

OAUTH

Google Web client

A Google Web application OAuth client whose redirect URI is https://<apiHost>/auth/callback.

Helm install

Install (or upgrade) the chart into its own namespace, pointing it at your registry, domains, bucket, and Secret:

helm upgrade --install drop infra/helm/drop \
  --namespace drop --create-namespace \
  --set image.repository=<acct>.dkr.ecr.<region>.amazonaws.com/drop \
  --set image.tag=0.1.0 \
  --set baseDomain=drop.example.com --set apiHost=api.drop.example.com \
  --set config.s3Bucket=drop-sites --set config.s3Region=<region> \
  --set config.allowedDomains=example.com \
  --set googleClientId=<id>.apps.googleusercontent.com \
  --set secret.existingSecret=drop-secrets \
  --set serviceAccount.annotations."eks\.amazonaws\.com/role-arn"=arn:aws:iam::<acct>:role/drop-s3 \
  --set ingress.className=alb \
  --set ingress.annotations."alb\.ingress\.kubernetes\.io/certificate-arn"=<acm-arn>

Then point DNS at the ingress / ALB:

RecordResolves toServes
api.<baseDomain>ingress / ALBControl plane, dashboard, OAuth callback (api)
*.<baseDomain>ingress / ALBEvery published site (edge, wildcard)
Note

Run helm lint / helm template to preview the rendered manifests, and see values.yaml for every knob. Keep the edge reachable only on the internal network. The install above is static-only — see Compute plane to enable the PaaS.

Compute plane (full PaaS)

By default compute.enabled=false and Drop is a static-site host (same feature set as the EC2 / ECS deploys). Set compute.enabled=true to add container apps (drop deploy), managed Postgres (drop db), and write-only app secrets. This is Kubernetes-only: it sets DROP_KUBECONFIG=in-cluster so the API drives the cluster through its pod ServiceAccount, and renders a ClusterRole granting that SA the RBAC to manage tenant namespaces + workloads.

Cluster operators required

Compute mode needs operators installed in the cluster first — the EKS Terraform does this when compute_enabled=true; locally infra/local/compute-up.sh does. They are: KEDA + the KEDA HTTP add-on (apps), CloudNativePG + the Barman Cloud Plugin (databases + backups), External Secrets Operator (only for compute.secretBackend=aws), and a gvisor RuntimeClass (untrusted-image sandbox).

helm upgrade --install drop infra/helm/drop \
  --namespace drop --create-namespace \
  ...                                   # all the static-only flags above, plus:
  --set compute.enabled=true \
  --set compute.secretBackend=kube \    # "kube" (no deps) or "aws" (via ESO)
  --set compute.imageBackend=registry \ # push tenant images to ECR
  --set compute.imageRegistry=<acct>.dkr.ecr.<region>.amazonaws.com/drop-apps \
  --set compute.blockedEgressCidrs="10.0.0.0/8\,172.20.0.0/16"
ValuePurpose
compute.enabledMaster switch. false = static-only; true = full PaaS (requires the operators above).
compute.secretBackendkube (app secrets as Kubernetes Secrets, no extra deps) or aws (AWS Secrets Manager via External Secrets).
compute.imageBackendregistry (push built app images to ECR — the prod default) or containerd.
compute.imageRegistryRegistry for tenant app images, e.g. <acct>.dkr.ecr.<region>.amazonaws.com/drop-apps. Keep separate from the platform-image repo. Required when imageBackend=registry.
compute.blockedEgressCidrsCluster pod + service CIDRs excluded from the tenant HTTPS egress allowlist (so tenant pods can't reach in-cluster services). Required on EKS.

Set on the API

For real login set DROP_DEV_AUTH=0. If the api egresses through a TLS-inspecting proxy, set NODE_EXTRA_CA_CERTS to the corp CA so it can reach the OIDC issuer. See Configuration for the full surface.

SSO (OIDC) setup

Login is generic OIDC and server-mediated: the api is the OAuth client, discovers the issuer, maps the claims, and issues its own session tokens, so the client secret lives only on the server — never on a developer's machine. Clients (CLI + MCP) need only DROP_API. Google is just the default issuer; point DROP_OIDC_ISSUER at any OIDC provider. One provider per deployment (multi-IdP is out of scope). Full per-provider recipes (Google, Okta, Microsoft Entra ID, Keycloak, Authentik) plus the break-glass admin and a throwaway local Keycloak are in SETUP_SSO.md.

  1. In your IdP, create a confidential / Web OAuth client using the Authorization Code flow.
  2. Register the redirect URI ${DROP_PUBLIC_URL}/auth/callback exactly (scheme, host, and path must match — e.g. https://api.drop.example.com/auth/callback).
  3. Set DROP_OIDC_ISSUER to the provider's issuer URL (leave unset for Google), and put the client id/secret in DROP_OIDC_CLIENT_ID / the Secret's DROP_OIDC_CLIENT_SECRET (or the legacy DROP_GOOGLE_* keys).
  4. Restrict sign-in with DROP_OIDC_ALLOWED_DOMAINS=example.com, and optionally a group gate (DROP_OIDC_GROUPS_CLAIM + DROP_OIDC_REQUIRED_GROUP).
Zero migration

Existing Google deployments are unchanged: the issuer defaults to Google and DROP_OIDC_CLIENT_ID/SECRET fall back to the legacy DROP_GOOGLE_CLIENT_ID/SECRET. Set the DROP_OIDC_* vars only to switch providers.

redirect_uri_mismatch

The URI registered in the IdP client must match DROP_PUBLIC_URL + /auth/callback character-for-character. If you change DROP_PUBLIC_URL, update the provider client too.

Edge caching & scaling

The edge is a stateless cache in front of S3 (the source of truth). It caches asset bytes on a per-pod directory set by DROP_EDGE_DISK_CACHE — in the chart an emptyDir (ephemeral node-local disk) mounted at /cache. Because it is just a cache, it needs no persistence and no sharing: per-pod + ephemeral is exactly right and scales to any replica count. A new or restarted pod re-warms from S3. The long-lived in-memory cache holds only the small per-site version pointer (10s TTL); asset bytes come from the disk cache (the OS page cache keeps hot files at RAM speed) or are fetched per-request from S3 and freed after the response.

KnobDefaultMeaning
edge.diskCache.sizeBytes5 GiBApp LRU cap → DROP_EDGE_DISK_CACHE_BYTES.
edge.diskCache.sizeLimit6GiemptyDir node-disk cap (a little above sizeBytes).

For autoscaling, set api.autoscaling.enabled / edge.autoscaling.enabled to turn on an HPA on CPU (api 2–6 replicas, edge 2–10 by default). CloudFront in front of the edge is optional.
